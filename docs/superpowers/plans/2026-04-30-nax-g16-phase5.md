## Task 1: Probe for `NAXFrag32::store_slice`

Write a standalone correctness probe that exercises `NAXFrag32::store_slice` on rectangular slice patterns. Use the same "inline NAXFrag32 source into the probe" pattern as `tools/probe_nax_frag32.py` (since `mx.fast.metal_kernel` cannot include source-tree headers).

**Files:**
- Create: `tools/probe_naxfrag32_store_slice.py`

**Test cases:**

| Slice | Expected |
|-------|----------|
| `start=(0,0), stop=(32,32)` | Full frag written; matches `store` |
| `start=(0,8), stop=(32,24)` | Cols 8..23 written, cols 0..7 and 24..31 untouched |
| `start=(4,0), stop=(28,32)` | Rows 4..27 written, rows 0..3 and 28..31 untouched |
| `start=(4,8), stop=(28,24)` | Both axes clipped |
| `start=(10,10), stop=(10,10)` | Empty slice — no writes |

For each case, allocate a 32×32 device buffer pre-filled with sentinel `-9999.0`. The probe writes the frag (filled with `r * 32 + c` values) via `store_slice`, then reads back. In-bounds positions must equal `r * 32 + c`; out-of-bounds positions must remain `-9999.0`.

The frag values are ints in `[-3, 3]` range, but for this probe we want unique-per-position values, so use `r * 32 + c` cast to `float`. Since the values are small integers, fp32 stores are exact.

- [ ] **Step 1: Replace the stub**

Edit `mlx/backend/metal/kernels/steel/gemm/nax.h`. Locate the stub (`store_slice` near line 943) and replace lines 935-949 with:

```cpp
  // Store a 32x32 frag to device memory, writing only elements inside
  // the rectangular slice [start_x, stop_x) x [start_y, stop_y). Out-of-
  // bounds device positions are NOT touched. Caller must allocate
  // `threadgroup T scratch[32*32]` and pass it.
  //
  // The signature mirrors BaseNAXFrag::store_slice so NAXTile can
  // dispatch with a single call site (gated on kPacking).
  template <typename T, typename U, typename StrX, typename StrY,
            typename StartX, typename StopX, typename StartY, typename StopY,
            typename OffX = Int<0>, typename OffY = Int<0>>
  METAL_FUNC static void store_slice(
      const thread dtype_frag_t<T>& src,
      device U* dst,
      StrX str_x,
      StrY str_y,
      StartX start_x,
      StopX stop_x,
      StartY start_y,
      StopY stop_y,
      threadgroup T* scratch,
      OffX off_x = Int<0>{},
      OffY off_y = Int<0>{}) {
    // Stage the 32x32 frag to threadgroup scratch via the contiguous store.
    store(src, scratch, /*ld=*/short(32));
    threadgroup_barrier(mem_flags::mem_threadgroup);

    // Per-thread bounded copy: each lane owns kElemsPerFrag (=32) elements.
    // Address them by flat index: flat = lane * 32 + i, r = flat / 32,
    // c = flat % 32. Apply the off_x/off_y shift to the slice bounds (which
    // are expressed in NAXTile coordinates, not raw frag coordinates).
    const ushort lane = __metal_get_thread_index_in_simdgroup(ushort());
    STEEL_PRAGMA_UNROLL
    for (short i = 0; i < kElemsPerFrag; ++i) {
      const short flat = lane * kElemsPerFrag + i;
      const short r = flat / 32;
      const short c = flat % 32;
      // r, c are within the 32x32 frag. Translate to NAXTile coords by adding
      // off_x / off_y (which are already in tile coordinates).
      const short tile_r = r + short(off_x);
      const short tile_c = c + short(off_y);
      if (tile_r >= short(start_x) && tile_r < short(stop_x) &&
          tile_c >= short(start_y) && tile_c < short(stop_y)) {
        dst[r * str_x + c * str_y] = static_cast<U>(scratch[r * 32 + c]);
      }
    }
  }
```

> Notes:
> - We use `r * str_x + c * str_y` for the device address (matching BaseNAXFrag's pattern at line 305). `str_y` is typically `Int<1>{}`, but we don't hardcode that.
> - The slice bounds compare against `tile_r/tile_c = r + off_x, c + off_y`. This is identical to the `r >= stop_x - sc.y` style in BaseNAXFrag (line 340) but uses NAXFrag32's actual per-lane coordinates rather than the regular per-thread offsets.
> - Wait — BaseNAXFrag's bound check at line 340 uses `(r + off_x) >= stop_x - sc.y` style (subtracting the cooperative-tensor base coord `sc`). For NAXFrag32, the staged-store route already places the frag in scratch as if its top-left were (0,0), so we don't need `sc`. Verify by inspection: NAXFrag32's `get_coord()` returns the cooperative-tensor base coord, and `store(src, scratch, 32)` places frag element (r,c) at `scratch[r*32 + c]`. The slice bounds in NAXTile::store_slice are passed as `start.y, stop.y, start.x, stop.x` where `start/stop` are in NAXTile coords (line 1363-1366). For a single-frag tile, off_x/off_y are zero. For multi-frag tiles, off_x/off_y advance by `idx_row * kFragRows` and `idx_col * kFragCols` (line 1367-1368). So `tile_r = r + off_x` is the correct NAXTile coord — no `sc` adjustment needed because the staging via scratch already collapses the per-lane layout.

- [ ] **Step 1: Write the sweep**

```python
"""
Sweep gather_mm shapes that exercise each bm path on g16:
  bm=64  (M/E > 48)
  bm=32  (24 < M/E <= 48)
  bm=16  (M/E <= 24, falls through to non-NAX gather_mm_rhs on g16)

Each shape compares mx.gather_mm against a numpy reference at fp32 rtol.
"""
import sys
import mlx.core as mx
import numpy as np


def run(M, N, K, E, name):
    rng = np.random.default_rng(0)
    a_np = rng.standard_normal((M, K), dtype=np.float32)
    b_np = rng.standard_normal((E, K, N), dtype=np.float32)
    rhs_np = rng.integers(0, E, size=(M,), dtype=np.uint32)

    a = mx.array(a_np)
    b = mx.array(b_np)
    rhs = mx.array(rhs_np)

    out = mx.gather_mm(a, b, rhs_indices=rhs, sorted_indices=True)
    out_np = np.array(out)

    expected = np.zeros((M, N), dtype=np.float32)
    for i in range(M):
        expected[i] = a_np[i] @ b_np[rhs_np[i]]

    err = np.max(np.abs(out_np - expected))
    rel_err = err / max(np.max(np.abs(expected)), 1e-9)
    print(f"  {name}: M={M}, N={N}, K={K}, E={E}, M/E={M/E:.1f}, "
          f"max|err|={err:.4e}, rel={rel_err:.4e}")
    if rel_err >= 1e-4:
        print(f"FAIL")
        sys.exit(1)


if __name__ == "__main__":
    print("Gather sweep across bm paths")
    # bm=64 path: M/E > 48
    run(M=512, N=128, K=128, E=8,  name="bm64_balanced")
    run(M=512, N=128, K=256, E=4,  name="bm64_large_k")
    # bm=32 path: 24 < M/E <= 48
    run(M=256, N=128, K=128, E=8,  name="bm32_balanced")  # M/E=32
    run(M=192, N=128, K=128, E=8,  name="bm32_lower")      # M/E=24
    # bm=16 path: M/E <= 24 (falls back to non-NAX on g16)
    run(M=128, N=128, K=128, E=8,  name="bm16_decode")     # M/E=16
    run(M=64,  N=128, K=128, E=8,  name="bm16_small")      # M/E=8
    run(M=8,   N=128, K=128, E=8,  name="bm16_minimal")    # M/E=1
    print("All sweep shapes passed")
```
