# NAX g16 — Phase 2 implementation plan (steel_gemm_fused integration)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Re-enable NAX on Apple M4 Pro (`applegpu_g16s`) for **dense fused matmul only**, by routing `steel_gemm_fused_nax` through the Phase-1 `NAXFrag32` (32×32 frag, `(32, 32, 32)` descriptor). Other NAX-using paths (splitk, gather, segmented, qmm, SDPA) remain gated off on g16 until later phases.

**Architecture:** NAXFrag32 owns its layout, `mma`, and the safe/rows staging (Phase-1 internalized this). NAXTile dispatches on `kPacking` and forwards a kernel-allocated `threadgroup T* scratch` to NAXFrag32 for safe/rows; for aligned `load`/`store` it computes per-frag base pointers and calls new device-pointer overloads we add in Task 2.0. The kernel allocates scratch under `if constexpr (NAXFrag_::kPacking == 1)`. The host dispatcher appends `_g16` to the kernel name when `nax_arch_flavor() == kG16`.

**Tech Stack:** C++ / Metal Shading Language, Apple `MetalPerformancePrimitives` (`mpp::tensor_ops::matmul2d`, `mpp::tensor_inline`), MLX C++ backend, Python tests via `mlx.core`, CMake. Hardware: Apple M4 Pro (`applegpu_g16s`).

**Spec:** `docs/superpowers/specs/2026-04-28-nax-g16-phase2-design.md` (commit `b07c6a63`).

**Branch:** `nax-g16-phase2`.
**Worktree:** `~/.config/superpowers/worktrees/mlx/nax-g16-phase2` — all paths in this plan are relative to that worktree.

## Reconciliation with Phase 1

Phase 1 already landed `NAXFrag32::load_safe` / `load_rows` / `store_safe` / `store_rows` (nax.h lines 732–822, commit `47379949`) — including the `threadgroup T* scratch` parameter and the row-per-lane staging. The original Phase 2 design assumed staging would happen at the NAXTile layer; in reality it's already inside NAXFrag32. This plan therefore has NAXTile **forward** scratch to NAXFrag32 rather than staging itself.

The Phase-1 safe methods are not yet hardware-validated — no probe in `tools/probe_nax_frag32.py` exercises them. Task 2.0b adds those probes before NAXTile relies on them.

What's still missing on NAXFrag32:
- A `load(device const U*, short ld)` overload (only the threadgroup variant exists). Needed by NAXTile::load (aligned).
- A `store(device U*, short ld)` overload. Needed by NAXTile::store (aligned) and by gemm_epilogue's aligned C-load → addmm → D-store path.
- Hardware probes for the existing safe/rows methods.

What's still missing on NAXTile:
- All five TODO-marked methods (`load_safe`, `load_rows`, `store_safe`, `store_rows`, `store_slice`) need a `kPacking == 1` dispatch + a `threadgroup T* scratch` param.
- The aligned `load` and `store` methods (lines 1051 and 1066) **also** need a `kPacking == 1` dispatch — their current bodies pass arguments to `NAXFrag_t::load`/`store` that NAXFrag32 doesn't accept (offset/stride args), so any `NAXTile<..., NAXFrag32>` instantiation would fail to compile.

What's still missing on the kernel + host side:
- `steel_gemm_fused_nax.h` is hardcoded for `BaseNAXFrag` (TM = SM/16, no scratch alloc, no kPacking==1 epilogue path).
- No `_g16` kernel instantiations exist.
- `is_nax_available()` is gated `gen >= (arch == 'p' ? 18 : 17)`.
- `matmul.cpp` has no flavor concept.
- Non-fused NAX paths (splitk, gather, segmented, qmm, SDPA) have no g16 escape gate.

### Task 2.0: Add `NAXFrag32::load(device)` and `store(device)` overloads

**Why:** NAXTile's aligned `load`/`store` paths and `gemm_epilogue`'s aligned C-load both pass a `device U*`. NAXFrag32 currently only accepts `threadgroup U*`. We add device-pointer twins of the existing threadgroup overloads (same body, different address space). `tensor_inline` works on both address spaces, and Phase-1's `tools/probe_nax_descriptor.py` row `(32, 32, 32, False)` proved 0 max|err| with a device-backed `tensor_inline` view — so this is a known-safe addition.

**Files:**
- Modify: `mlx/backend/metal/kernels/steel/gemm/nax.h` (inside `struct NAXFrag32`, just after the existing threadgroup `store` at line ~720)

- [ ] **Step 1: Add `load(device, ld)` after the existing threadgroup `load`**

In `mlx/backend/metal/kernels/steel/gemm/nax.h`, after the closing `}` of the threadgroup `load` (line ~692), insert:

```cpp
  // Device-memory variant of the contiguous load. Same body as the
  // threadgroup overload above with the address space swapped — used by
  // NAXTile's aligned device-pointer load and by gemm_epilogue's aligned
  // C-load. Validated by tools/probe_nax_frag32.py::test_mma_via_dv_load_store.
  template <typename T, typename U>
  METAL_FUNC static void load(
      thread dtype_frag_t<T>& dst,
      const device U* src,
      const short ld) {
    constexpr auto desc = mpp::tensor_ops::matmul2d_descriptor(
        32, 32, 32,
        /*transpose_left=*/false,
        /*transpose_right=*/false,
        /*relaxed_precision=*/true,
        mpp::tensor_ops::matmul2d_descriptor::mode::multiply_accumulate);
    mpp::tensor_ops::matmul2d<desc, metal::execution_simdgroup> op;
    auto ct = op.template get_left_input_cooperative_tensor<T, T, T>();

    metal::dextents<int32_t, 2> ext(32, ld);
    metal::tensor<device U, metal::dextents<int32_t, 2>, metal::tensor_inline>
        view(const_cast<device U*>(src), ext);
    ct.load(view);

    STEEL_PRAGMA_UNROLL
    for (short i = 0; i < kElemsPerFrag; i++) {
      dst[i] = static_cast<T>(ct[i]);
    }
  }
```

### Task 2.2: NAXTile dispatch on `kPacking == 1` (six methods + `store_slice`)

**Why:** With NAXFrag32's per-thread layout being non-uniform, NAXTile's existing methods can't pass `(off_x, off_y)` arithmetic args to `NAXFrag_t::load` / `store` — those args are meaningless for NAXFrag32. We add an `if constexpr (NAXFrag_t::kPacking == 1)` branch that computes per-frag base pointers, forwards `threadgroup T* scratch` for safe variants, and calls the matching NAXFrag32 method. BaseNAXFrag callers stay unchanged via the default-`nullptr` scratch param.

```cpp
template <
    bool kAlignedM,
    bool kAlignedN,
    class NAXTile_t,
    typename T>
void gemm_epilogue(
    thread NAXTile_t& Dtile,
    const device T* C,
    const constant GEMMParams* params,
    const constant GEMMAddMMParams* addmm_params,
    const short sgp_sm,
    const short sgp_sn,
    threadgroup typename NAXTile_t::elem_type* scratch = nullptr) {

  (void)params;

  using V = typename NAXTile_t::elem_type;

  constexpr short TM = NAXTile_t::kTileRows;
  constexpr short TN = NAXTile_t::kTileCols;
  constexpr short kElemsPerFrag = NAXTile_t::kElemsPerFrag;

  using CFrag = typename NAXTile_t::NAXFrag_t;
  using cfrag_t = typename CFrag::template dtype_frag_t<T>;

  static_assert(
      sizeof(V) >= sizeof(T),
      "gemm_epilogue: AccumType must be >= sizeof(T) so the shared scratch "
      "buffer can be safely cast to threadgroup T* for the C load.");

  const_for_loop<0, TM, 1>([&](auto mm) {
    const_for_loop<0, TN, 1>([&](auto nn) {
      auto m = mm * Int<CFrag::kFragRows>{};
      auto n = nn * Int<CFrag::kFragCols>{};

      cfrag_t celems;

      if constexpr (CFrag::kPacking == 1) {
        // NAXFrag32 path: ld for the C view is the row stride (ldc) since C is
        // row-major. fdc is the inner-dim element stride; with row-major C and
        // float accum, fdc must be 1 — assert that since NAXFrag32::load_safe
        // treats the buffer as contiguous along the column dimension.
        // (If a real test_blas case violates this, fall back to staging.)
        if constexpr (kAlignedM && kAlignedN) {
          CFrag::load(
              celems,
              C + m.value * addmm_params->ldc + n.value * addmm_params->fdc,
              short(addmm_params->ldc));
        } else {
          CFrag::load_safe(
              celems,
              C + m.value * addmm_params->ldc + n.value * addmm_params->fdc,
              addmm_params->ldc,
              sgp_sm - m.value,
              sgp_sn - n.value,
              (threadgroup T*)scratch);
        }
      } else {
        // existing kPacking==2 body, unchanged.
        if constexpr (kAlignedM && kAlignedN) {
          CFrag::load(celems, C, addmm_params->ldc, addmm_params->fdc, m, n);
        } else {
          CFrag::load_safe(
              celems, C, addmm_params->ldc, addmm_params->fdc,
              sgp_sm, sgp_sn, m, n);
        }
      }

      thread auto& delems = Dtile.template frag_at<mm, nn>();

      STEEL_PRAGMA_UNROLL
      for (short i = 0; i < kElemsPerFrag; i++) {
        if (do_axpby) {
          delems[i] = addmm_params->alpha * delems[i] +
              addmm_params->beta * static_cast<V>(celems[i]);
        } else {
          delems[i] += static_cast<V>(celems[i]);
        }
      }
    });
  });
}
```
