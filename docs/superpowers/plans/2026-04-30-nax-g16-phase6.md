## Task 1: Diagnostic probe — pin down the broken layer

Build a focused isolation probe that exercises `NAXFrag32::mma` with all 4 transpose configurations and identifies which layer is broken. The probe stays in tree as a permanent regression test after the fix lands.

**Files:**
- Create: `tools/probe_naxfrag32_transpose.py`

**Reference patterns** (read these first):
- `tools/probe_nax_frag32.py` — HEADER block + run_case structure. Copy the HEADER conventions verbatim.
- `tools/probe_naxfrag32_store_slice.py` — the most recent probe; uses `Int<N>` helper and the `_NAX_STRUCT_MARKER`-based HEADER splitting.
- `mlx/backend/metal/kernels/steel/gemm/nax.h:617-657` — `NAXFrag32::mma` signature and body. The probe will call this directly.
- `mlx/backend/metal/kernels/steel/gemm/nax.h:667-692` — `NAXFrag32::load` (threadgroup overload). The hardcoded `transpose_left=false, transpose_right=false` descriptor at lines 671-676 is the suspect.

**Probe scope:**

1. **Main probe — `test_mma_all_transposes`**: 32×32 fp32 inputs A and B (small integer values in [-3, 3] so reductions are exact). For each `(transpose_a, transpose_b) ∈ {(F,F), (F,T), (T,F), (T,T)}`:
   - Inside the kernel: load A and B into NAXFrag32 frags via the existing `NAXFrag32::load`.
   - Call `NAXFrag32::mma(C, A, bool_constant<transpose_a>{}, B, bool_constant<transpose_b>{})`.
   - Store C via `NAXFrag32::store`.
   - Python: compare against `numpy.dot(A_or_AT, B_or_BT)`.
   - On main: expect `(F,F)` to pass and the three transposed cases to fail. The probe asserts each case independently.

2. **Layer-isolation sub-probe — `test_mma_with_role_aware_load`**: same 4 cases, but each load uses a NEW load helper (defined inline in the probe HEADER) that takes a `(role, transpose)` and uses the matching descriptor when constructing the cooperative_tensor. If this fixes the transposed cases → layer 1. If it doesn't → layer 2 or 3 (the load was already correct; mma or descriptor is wrong).

The two sub-probes share the input-generation harness; the difference is which `load` is invoked inside the kernel.

```python
def test_mma_all_transposes():
    np.random.seed(0)
    A = np.random.randint(-3, 4, size=(32, 32)).astype(np.float32)
    B = np.random.randint(-3, 4, size=(32, 32)).astype(np.float32)

    cases = [
        ("FF", False, False, A @ B),
        ("FT", False, True,  A @ B.T),
        ("TF", True,  False, A.T @ B),
        ("TT", True,  True,  A.T @ B.T),
    ]

    failures = []
    for name, ta, tb, expected in cases:
        # Build a kernel per (ta, tb) — the transpose flags are template constants.
        kernel = mx.fast.metal_kernel(
            name=f"probe_naxfrag32_mma_{name}",
            input_names=["a_in", "b_in"],
            output_names=["out_buf"],
            header=FULL_HEADER,
            source=_KERNEL_BODY_MAIN.format(ta=str(ta).lower(), tb=str(tb).lower()),
        )
        a_mx = mx.array(A); b_mx = mx.array(B)
        out = kernel(
            inputs=[a_mx, b_mx],
            output_shapes=[(32, 32)],
            output_dtypes=[mx.float32],
            grid=(32, 1, 1),
            threadgroup=(32, 1, 1),
        )[0]
        got = np.array(out)
        err = np.max(np.abs(got - expected))
        ok = err < 0.5
        status = "OK" if ok else "FAIL"
        print(f"  {status}  test_mma_all_transposes[{name}]  max|err|={err:.4e}")
        if not ok:
            failures.append((name, ta, tb, err))
    return failures
```

- [ ] **Step 4: Add the layer-isolation sub-probe `test_mma_with_role_aware_load`**

Inside `FULL_HEADER`, add a helper struct or namespace `NAXFrag32_RoleAware` (or use a free function) that exposes a `load_with_descriptor` template:

```cpp
// Layer-1 isolation helper: load with the descriptor that matches the
// (role, transpose) the consumer's mma will use. If layer 1 is the bug,
// this load makes the transposed cases pass.
template <bool transpose_left, bool transpose_right, bool is_right_input,
          typename T, typename U>
METAL_FUNC void naxfrag32_load_role_aware(
    thread NAXFrag32::dtype_frag_t<T>& dst,
    const threadgroup U* src,
    const short ld) {
  constexpr auto desc = mpp::tensor_ops::matmul2d_descriptor(
      32, 32, 32,
      transpose_left, transpose_right,
      true,
      mpp::tensor_ops::matmul2d_descriptor::mode::multiply_accumulate);
  mpp::tensor_ops::matmul2d<desc, metal::execution_simdgroup> op;
  auto ct = is_right_input
      ? op.template get_right_input_cooperative_tensor<T, T, T>()
      : op.template get_left_input_cooperative_tensor<T, T, T>();
  metal::dextents<int32_t, 2> ext(32, ld);
  metal::tensor<threadgroup U, metal::dextents<int32_t, 2>, metal::tensor_inline>
      view(const_cast<threadgroup U*>(src), ext);
  ct.load(view);
  STEEL_PRAGMA_UNROLL
  for (short i = 0; i < NAXFrag32::kElemsPerFrag; i++) {
    dst[i] = static_cast<T>(ct[i]);
  }
}
```

Note: ternary on a `constexpr if` style is cleaner. If Metal's `?:` doesn't dispatch correctly here, use `if constexpr`.

The sub-probe kernel body uses `naxfrag32_load_role_aware<ta, tb, false>(A_frag, A_buf, 32)` for A and `naxfrag32_load_role_aware<ta, tb, true>(B_frag, B_buf, 32)` for B.

Replace the existing function with a parameterized version:

```cpp

METAL_FUNC static void load(
    thread dtype_frag_t<T>& dst,
    const threadgroup U* src,
    const short ld) {
  constexpr auto desc = mpp::tensor_ops::matmul2d_descriptor(
      32, 32, 32,

      /*transpose_right=*/ (role == Role::Right) ? transpose : false,
```

Update to:

```cpp
template <Role role, bool transpose, typename U>
METAL_FUNC void load(const device U* src, const int ld,
                     threadgroup elem_type* scratch) {
  if constexpr (NAXFrag_t::kPacking == 1) {
    const_for_loop<0, kTileRows, 1>([&](auto idx_row) {
      const_for_loop<0, kTileCols, 1>([&](auto idx_col) {
        constexpr short m_off = idx_row.value * kFragRows;
        constexpr short n_off = idx_col.value * kFragCols;
        NAXFrag_t::template load_rows<role, transpose>(
            frag_at<idx_row.value, idx_col.value>(),
            src + m_off * ld + n_off,
            ld,
            short(kFragRows),
            scratch);
      });
    });
  } else {
    const_for_loop<0, kTileRows, 1>([&](auto idx_row) {
      const_for_loop<0, kTileCols, 1>([&](auto idx_col) {
        NAXFrag_t::load(
            frag_at<idx_row.value, idx_col.value>(),
            src,
            ld,
            Int<1>{},
            idx_row * Int<kFragRows>{},
            idx_col * Int<kFragCols>{});
      });
    });
  }
}
```

If `test_mma_with_role_aware_load` doesn't fix the transposed cases, the bug is in `NAXFrag32::mma` itself — the element-routing logic that copies from `dtype_frag_t` to `cooperative_tensor` doesn't account for transpose.

This task is **diagnostic-driven**: design a smaller probe that loads operands with KNOWN positional values and inspects what arrives in `ct_b` after the load. Compare for transpose=false vs transpose=true. The discrepancy reveals the right element permutation.

The implementation plan for the layer-2 fix can't be predesigned without the diagnostic data. The implementer should:
1. Build a "single-element-marker" probe that loads B with one element set to a known sentinel and zeros elsewhere.
2. Run the mma; observe where the sentinel ended up in C.
3. Cross-tabulate sentinel position vs C position for transpose=false and transpose=true.
4. Infer the permutation; apply it to the `ct_b[i] = B[i]` (and similarly `ct_a[i] = A[i]`) loops in `NAXFrag32::mma`.

If after probing it becomes clear the bug is intrinsic to MPP's `matmul2d` on g16 (descriptor itself), escalate to Task 2c.

## Task 2c: Layer-3 gate (only if both layer 1 and layer 2 fail)
