# NAX-on-g16 Phase 9 — matmul attribution + targeted fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Attribute the unresolved ~30% NAX matmul tax on g16 to one of three named cost contributors (mma dispatch, `ct.load`, per-iter coop_tensor setup) by building three diagnostic kernel variants (V1 no-mma, V2 zero-load, V3 hoisted-op), benching against the V0 baseline on two diagnostic shapes, and computing per-contributor cost. If per-iter setup dominates, ship the hoisted-op fix in the real `tile_matmad_nax` and reach ≥0.95× of non-NAX on `gemm_splitk M=64 N=64 K=8192` without regressing other cases by >0.05×; otherwise ship the report and a Phase 10 spec stub.

**Architecture:** Single PR with sequential gates. A new `MLX_NAX_DIAG_VARIANT` macro in `nax_common.h` selects the active variant at build time (0=baseline, 1=no-mma, 2=zero-load, 3=hoisted-op). The engineer rebuilds MLX once per variant, runs the bench harness in a new `--diag` mode (filters to the two diagnostic shapes), saves the per-variant output, and computes the attribution table by hand from the four reports. Decision rule (per spec) chooses between same-phase fix and Phase 10 deferral. A2 Instruments is a manual confirmation step only invoked if V1–V3 numbers are ambiguous.

**Tech Stack:** Python 3 + `mlx.core` (bench harness), Metal C++ (variant gates and hoisted-op refactor in `nax_common.h`), Apple Instruments / Metal System Trace (manual, conditional).

**Spec:** `docs/superpowers/specs/2026-05-02-nax-g16-phase9-design.md` (commit `0683e53e` on branch `nax-g16-phase9-design`).

**Python interpreter:** `/Users/daniel/mlx/.venv/bin/python` (system `python3` lacks `mlx.core`).

**Worktree:** Run in a fresh worktree branched from `nax-g16-phase9-design` (e.g., `/Users/daniel/mlx-worktrees/nax-g16-phase9` on branch `nax-g16-phase9`). The `nax-g16-phase9-design` branch already contains the spec.

```python
def run_child(arm: str, disable_nax: bool, capture_dir: str | None = None,
              diag_label: str | None = None) -> tuple[dict, list[dict]]:
    env = dict(os.environ)
    if disable_nax:
        env["MLX_DISABLE_NAX"] = "1"
    else:
        env.pop("MLX_DISABLE_NAX", None)
    if capture_dir is not None:
        env["MLX_NAX_BENCH_CAPTURE_DIR"] = capture_dir
    else:
        env.pop("MLX_NAX_BENCH_CAPTURE_DIR", None)
    if diag_label is not None:
        env["MLX_NAX_BENCH_DIAG_LABEL"] = diag_label
    else:
        env.pop("MLX_NAX_BENCH_DIAG_LABEL", None)
    cmd = [sys.executable, __file__, "--child", arm]
    proc = subprocess.run(cmd, env=env, capture_output=True, text=True)
    if proc.returncode != 0:
        sys.stderr.write(proc.stderr)
        raise SystemExit(f"child arm={arm} failed (exit {proc.returncode})")
    header = {}
    cases = []
    for line in proc.stdout.splitlines():
        line = line.strip()
        if not line:
            continue
        rec = json.loads(line)
        if rec.get("_header"):
            header = rec
        else:
            cases.append(rec)
    return header, cases
```
